# DistillationDPO 环境配置：问题与解决方案

本文档记录在 Ubuntu 24.04、CUDA 11.1、Python 3.8 环境下配置 DistillationDPO（含 MinkowskiEngine）时遇到的问题及对应解决办法。

---

## 1. CUDA 11.1 安装与路径

### 问题
- 直接运行 `cuda_11.1.0_455.23.05_linux.run` 时提示 `No such file or directory`，说明当前目录下没有该安装包。
- 将 run 包下载到 `/mnt/data` 时出现 `Permission denied`，当前用户对 `/mnt/data` 无写权限。

### 解决方案
- 将 run 包下载到有写权限的目录（如家目录）：
  ```bash
  cd ~
  wget -O ~/cuda_11.1.0_455.23.05_linux.run https://developer.download.nvidia.com/compute/cuda/11.1.0/local_installers/cuda_11.1.0_455.23.05_linux.run
  ```
- 安装时使用**完整路径**指定 run 文件，并将 CUDA 安装到数据盘以节省根分区空间：
  ```bash
  sudo sh ~/cuda_11.1.0_455.23.05_linux.run --toolkit --silent --override --installpath=/mnt/data/cuda-11.1
  ```
- 若希望安装到系统路径，可省略 `--installpath`，默认会安装到 `/usr/local/cuda-11.1`。

---

## 2. GCC 版本与 CUDA 11.1 不兼容

### 问题
- CUDA 11.1 的 nvcc 仅支持 GCC ≤ 10，而 Ubuntu 24.04 默认 GCC 为 13。
- 编译 MinkowskiEngine 时报错：
  ```text
  unsupported GNU version! gcc versions later than 10 are not supported!
  ```

### 解决方案
- 安装 GCC 10 并指定为编译 MinkowskiEngine 时的 C/C++ 编译器：
  ```bash
  sudo apt update && sudo apt install -y gcc-10 g++-10
  export CC=gcc-10 CXX=g++-10
  ```
- 在编译 MinkowskiEngine 前务必设置 `CC` 和 `CXX`（见下文完整编译流程）。

---

## 3. 系统头文件与 nvcc 不兼容（`__int128`）

### 问题
- 使用 gcc-10 后，nvcc 在编译时会间接包含 `/usr/include/linux/types.h`。
- 该头文件在 `#ifdef __SIZEOF_INT128__` 块中定义了 `__s128`/`__u128`（基于 `__int128`），CUDA 11.1 的 nvcc 无法正确解析，报错：
  ```text
  /usr/include/linux/types.h:12:27: error: expected initializer before ‘__s128’
  ```

### 解决方案
- 在 MinkowskiEngine 的构建中为 nvcc 增加 `-U__SIZEOF_INT128__`，使上述 `#ifdef` 块不被编译，从而避免 `__int128` 相关定义。
- 已在本项目内的 `scripts/MinkowskiEngine-0.5.4/setup.py` 中在 `NVCC_FLAGS` 里加入该选项，从该目录编译即可生效。

---

## 4. pip 误报 “MinkowskiEngine 已满足”

### 问题
- 执行 `pip install MinkowskiEngine==0.5.4` 时提示 “Requirement already satisfied: MinkowskiEngine==0.5.4 in /mnt/data/DistillationDPO”，但实际 `import MinkowskiEngine` 失败。
- 原因：pip 把项目目录当成了已安装的 MinkowskiEngine，并未真正安装带 CUDA 的 C++ 扩展。

### 解决方案
- 必须从**源码**编译安装 MinkowskiEngine（使用项目内已修补的 `scripts/MinkowskiEngine-0.5.4`），并设置好 `CUDA_HOME`、`CC`、`CXX` 等（见下文完整流程）。

---

## 5. 环境变量与编译/运行

编译 MinkowskiEngine 以及运行 DistillationDPO 时，都需要正确设置 CUDA 与编译器环境变量。

### 设置环境变量（每次新开终端或跑训练/推理前建议执行）

```bash
# CUDA 11.1 路径（若安装到 /usr/local/cuda-11.1 则改为该路径）
export CUDA_HOME=/mnt/data/cuda-11.1
export PATH="$CUDA_HOME/bin:$PATH"
export LD_LIBRARY_PATH="$CUDA_HOME/lib64:$LD_LIBRARY_PATH"

# 仅编译 MinkowskiEngine 时需要
export CC=gcc-10 CXX=g++-10

# 激活虚拟环境
source /mnt/data/DistillationDPO/.venv38/bin/activate

# 可选：避免 MinkowskiEngine 的 OMP 警告（数值可按 CPU 核心数调整）
export OMP_NUM_THREADS=16
```

### 一次性写入 shell 配置（可选）

若希望登录时自动生效，可将上述 `export` 行（不含 `CC`/`CXX`，除非需要经常重新编译）追加到 `~/.bashrc` 或 `~/.profile`，例如：

```bash
# DistillationDPO / CUDA 11.1
export CUDA_HOME=/mnt/data/cuda-11.1
export PATH="$CUDA_HOME/bin:$PATH"
export LD_LIBRARY_PATH="$CUDA_HOME/lib64:$LD_LIBRARY_PATH"
```

## 6. MinkowskiEngine 从源码安装完整流程（总结）

在已安装 CUDA 11.1、gcc-10 且已应用 `setup.py` 中 `-U__SIZEOF_INT128__` 的前提下：

```bash
# 1. 环境变量
export CUDA_HOME=/mnt/data/cuda-11.1
export PATH="$CUDA_HOME/bin:$PATH"
export LD_LIBRARY_PATH="$CUDA_HOME/lib64:$LD_LIBRARY_PATH"
export CC=gcc-10 CXX=g++-10

# 2. 进入项目并激活 venv
cd /mnt/data/DistillationDPO
source .venv38/bin/activate

# 3. 从项目内修补版源码编译安装
cd scripts/MinkowskiEngine-0.5.4
rm -rf build *.egg-info
python setup.py install --blas=openblas
cd ../..

# 4. 验证
python -c "import torch; import MinkowskiEngine as ME; print('torch:', torch.__version__, 'CUDA:', torch.cuda.is_available()); print('MinkowskiEngine OK')"
```
